RATE LIMITING (Built-in)
═══════════════════════════════════════════════════════════════════════════════

Automatic enforcement:
  • 1 symbol every 2 seconds (per-symbol limit)
  • 30 symbols per minute (global limit)
  • Automatic delays — no manual throttling needed

Practical performance:
  • 5 symbols:  ~10 seconds
  • 10 symbols: ~20 seconds
  • 20 symbols: ~40 seconds
  • 30+ symbols: progressively slower (hits minute limit)

Recommendation:
  • UI fetches: use 5-10 symbols for responsive interface
  • Nightly batches: can go up to 30+ symbols (runs overnight)


TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Q: TradingView tab doesn't appear
A: tvDatafeed not installed → run: uv pip install tvDatafeed --break-system-packages

Q: "Fetch Data" fails with error
A: Check tvDatafeed installation & symbol validity on TradingView.com
   • NSE: use uppercase no suffix (RELIANCE, not RELIANCE.NS)
   • ASX: use uppercase no suffix (BHP, not BHP.AX)
   • US: use uppercase (AAPL, MSFT)

Q: Data loads very slowly (60+ seconds)
A: Normal! Rate limiting is 1 symbol per 2 seconds
   → Reduce to 5-10 symbols for faster results
   → Or wait patiently (system working correctly)

Q: CSV export not working
A: Ensure C:\StockData\ has write permissions
   → Try fetching data first (button must show data)
   → Verify no file locks on directory
